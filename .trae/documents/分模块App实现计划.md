## 分模块App实现计划

### 1. 安装依赖
- 安装 zustand 用于全局状态管理
- 安装 swr 用于请求管理

### 2. 创建全局状态管理
- 创建 `src/store` 目录
- 创建 `userStore.ts` 用于存储用户类型和密钥信息
- 实现密钥验证逻辑

### 3. 修改路由结构
- 修改 `app/_layout.tsx`，将入口页面改为密钥输入页
- 创建动态路由，根据用户类型导航到不同模块

### 4. 创建密钥输入页面
- 创建 `app/index.tsx` 作为密钥输入页面
- 实现密钥输入表单
- 实现密钥验证逻辑

### 5. 创建模块首页
- 创建 `app/moduleA/index.tsx` 作为模块A首页
- 创建 `app/moduleB/index.tsx` 作为模块B首页
- 可根据需要扩展更多模块

### 6. 实现导航逻辑
- 在密钥验证成功后，根据密钥类型导航到对应模块
- 使用 Expo Router 的动态导航功能

### 7. 配置 SWR
- 创建 `src/utils/fetcher.ts` 用于配置 SWR 请求
- 实现基本的请求封装

### 8. 测试与优化
- 测试不同密钥类型的导航逻辑
- 优化用户体验
- 确保状态管理正常工作

### 技术栈
- **状态管理**: zustand
- **请求管理**: swr
- **路由**: Expo Router
- **UI**: React Native 原生组件

### 实现步骤
1. 安装依赖
2. 创建状态管理 store
3. 创建密钥输入页面
4. 创建模块首页
5. 修改路由配置
6. 实现导航逻辑
7. 配置 SWR
8. 测试功能

### 预期效果
- 用户打开 app 后看到密钥输入页面
- 输入正确的密钥后，根据密钥类型进入对应模块首页
- 不同模块有独立的页面和功能
- 全局状态管理正常工作
- SWR 请求配置完成
